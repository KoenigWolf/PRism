# PRism Brand Guidelines v2.0

> PR × Prism — 競合のPR戦略を"分光"し、勝ちパターンを可視化する

---

## 0. このドキュメントについて

本書は、PRismブランドの一貫性と品質を維持するための公式ガイドラインである。デザイナー、エンジニア、マーケター、外部パートナーを含むすべてのブランド接点において、このガイドラインに準拠すること。

**対象者**: プロダクトチーム / マーケティング / 外部デザイナー / パートナー企業

**バージョン管理**: 本ガイドラインの更新権限はブランドオーナーに限定する。改定時は全チームへ通知し、旧バージョンからの移行期間を2週間設ける。

---

## 1. ブランドストーリー

### 1.1 ブランドの起源

光がプリズムを通過すると、目に見えなかった波長が虹として分離される。PRismは、PR業界に同じ体験を提供する。複雑に混在する競合のPR活動を、データとフレームワークの力で美しく"分光"し、隠れていた勝ちパターンを可視化する。

### 1.2 ネーミングの構造

```
  PR    +    ism    +    Prism
  ──         ───         ─────
 広報     主義・手法    光を分解する多面体
```

「PRism」は3つの意味を1語に圧縮したコインドネーム（造語）である。

| レイヤー | 要素 | 意味 | ブランドへの貢献 |
|---|---|---|---|
| **機能** | PR（Public Relations） | ドメイン領域 | 一目で「PR関連のツール」と認知される |
| **比喩** | Prism（プリズム） | 光の分離 | 複雑なPR情報をPESO区分で分光するメタファー |
| **哲学** | -ism（〜主義） | 方法論 | データドリブンPRという「新しい流儀」の提唱 |

### 1.3 ブランドミッション

> **PR戦略の"なぜ"を可視化し、すべてのPRパーソンに「次の一手」を届ける。**

### 1.4 ブランドビジョン

> **PR業界のインテリジェンス基盤となり、データで意思決定する文化を標準にする。**

### 1.5 ブランドバリュー

| バリュー | 定義 | 行動指針 |
|---|---|---|
| **Insight First** | 分析で終わらず、示唆を提供する | 「だから何？」に常に答える |
| **Clarity** | 複雑な情報を美しく構造化する | 情報を足すのではなく、ノイズを削る |
| **Confidence** | データに裏打ちされた自信で語る | 「〜かもしれない」ではなく「〜です」 |
| **Accessibility** | 誰でも使える価格とUI | 専門家だけのツールにしない |

## 2. 表記ルール

### 2.1 正式表記

| 用途 | 表記 | 備考 |
|---|---|---|
| **正式名称** | **PRism** | 先頭2文字「PR」のみ大文字。これが最も重要なルール |
| ロゴタイプ内 | PRism | フォントウェイト差でPRを強調 |
| 日本語併記 | PRism（プリズム） | 初出時のみカタカナを添える |
| URL / SNS | @prism_pr / prism-pr | 小文字統一＋「_pr」で領域明示 |
| コード・API | `prism` / `prism-pr` | 小文字ケバブケース |
| ハッシュタグ | #PRism #PRism_PR | SNS投稿時の公式タグ |

### 2.2 NG表記（厳守）

| NG表記 | 理由 |
|---|---|
| ~~PRISM~~ | NSAの監視プログラムを連想。絶対禁止 |
| ~~Prism~~ | PRの2文字が埋没し、ブランド認知が薄れる |
| ~~prism~~ | 固有名詞としての存在感がなくなる |
| ~~PR ism~~ | スペースで分断すると意味が壊れる |
| ~~PRIsm~~ | 大文字の境界が不正確 |
| ~~pRism~~ | いかなる変形も不可 |

### 2.3 文中での使用ルール

- 文頭・文中・文末を問わず、常に「PRism」と表記する（文頭でも「Prism」にしない）
- 複数形にしない（"PRisms" は不可）
- 動詞化しない（"PRism it" は不可）
- 他の単語と結合しない（"PRism-powered" は可。"PRismize" は不可）

## 3. タグライン

### 3.1 タグライン体系

ブランドの伝え方を場面ごとに最適化する3層構造。

| 層 | タグライン | 用途 | 言語 |
|---|---|---|---|
| **コア** | **「競合のPR、分光する。」** | ロゴ直下、名刺、LP Hero | 日本語 |
| **機能訴求** | 「勝ちパターンを、データで解く。」 | 広告、記事タイトル、セミナー | 日本語 |
| **差別化** | 「PR戦略の"なぜ"を、可視化する。」 | 競合比較、セールス資料 | 日本語 |
| **英語版** | "Decompose. Compare. Outsmart." | グローバル展開、LP英語版 | English |

### 3.2 タグライン使用ルール

- コアタグラインは改変不可。フォント・サイズ・色の変更のみ許可
- タグラインとロゴの間には最低ロゴ高さの50%の余白を確保
- タグラインを単独で使用する場合、必ずPRismロゴまたは名前が同一視野内にあること
- 翻訳時は意訳を推奨。直訳による違和感を避ける

## 4. ロゴシステム

### 4.1 ロゴ構成

PRismのロゴは3つの要素で構成される。

```
┌──────────────────────────────────────────────┐
│                                              │
│         ◇          ← シンボルマーク           │
│        /|\              （プリズム多面体）       │
│       / | \                                   │
│                                              │
│      PRism         ← ワードマーク              │
│                        （PR: Bold / ism: Light）│
│                                              │
│  ◇ PRism           ← 横組みロックアップ        │
│                                              │
└──────────────────────────────────────────────┘
```

| 要素 | 説明 |
|---|---|
| **シンボルマーク** | プリズム（三角多面体）から虹色の光線が分散するモチーフ。「分光」の視覚表現 |
| **ワードマーク** | Inter フォント。「PR」= Bold 700、「ism」= Light 300 |
| **ロックアップ** | シンボル＋ワードマークの一体型。横組み（水平）がデフォルト |

### 4.2 ロゴバリエーション

| バリエーション | 用途 | 背景条件 |
|---|---|---|
| **フルカラー（Primary）** | LP、資料、名刺 | 白背景・明るい背景 |
| **フルカラー（Reversed）** | ダークモードUI、暗い写真上 | 暗い背景 |
| **モノクロ（Black）** | 白黒印刷、FAX | 白背景 |
| **モノクロ（White）** | 暗い背景上での白抜き | 暗い背景 |
| **シンボルのみ** | ファビコン、アプリアイコン、SNSアバター | 正方形必須の場面 |

### 4.3 クリアスペース（保護領域）

ロゴの視認性を確保するため、ロゴ周囲に最低限の余白を設ける。

```
┌─────────────────────────────────┐
│         ↕ X                      │
│    ←X→ ◇ PRism ←X→              │
│         ↕ X                      │
└─────────────────────────────────┘

X = ワードマーク「P」の高さ
```

- クリアスペース内には、いかなるテキスト・図形・写真も配置しない
- 共同ブランディング時は2X以上の余白を推奨

### 4.4 最小使用サイズ

| 形式 | 最小幅 |
|---|---|
| フルロゴ（横組み） | 80px（デジタル） / 25mm（印刷） |
| シンボルのみ | 24px（デジタル） / 8mm（印刷） |

### 4.5 ロゴの禁止事項

以下の使用は厳禁とする。

| 禁止事項 | 理由 |
|---|---|
| ロゴの色を変更する | ブランドカラー以外の着色は一貫性を損なう |
| ロゴを変形・回転する | 認知の再現性が低下する |
| ロゴに影・グロー効果を追加する | デザインの品位を損なう |
| ロゴを柄や写真の上に直接配置する | 視認性が確保できない場合は不可 |
| ロゴの一部を切り取る | 完全な状態でのみ使用 |
| ロゴ要素の配置を変える | シンボルとワードマークの位置関係は固定 |

### 5.6 アクセシビリティ基準

PRismはWCAG 2.1 AA準拠を必須とする。

| 要件 | 基準 | PRismでの適用 |
|---|---|---|
| 通常テキスト | コントラスト比 4.5:1以上 | Slate 700 on White = 7.4:1 ✅ |
| 大テキスト（18px+） | コントラスト比 3:1以上 | Primary on White = 4.6:1 ✅ |
| UIコンポーネント | コントラスト比 3:1以上 | ボタン、入力フィールド等 |
| ダークモードテキスト | コントラスト比 4.5:1以上 | Slate 200 on Slate 900 = 11.3:1 ✅ |

**カラーブラインドネス対応**:

- PESO区分色は色覚多様性（P型・D型・T型）テスト済み
- 色だけで情報を伝えない。必ずラベル・アイコン・パターンを併用する
- グラフでは色に加えて線種（実線・点線・破線）を割り当てる

### 5.7 グラデーション

ブランドグラデーションはアクセント要素として限定使用する。

| 名前 | CSS | 用途 |
|---|---|---|
| **Prism Gradient** | `linear-gradient(135deg, #4F46E5, #8B5CF6)` | Hero背景、CTA強調 |
| **Spectrum** | `linear-gradient(90deg, #F59E0B, #3B82F6, #10B981, #8B5CF6)` | PESOスペクトラム表現、ローディング |

### 6.3 タイポグラフィルール

- 日本語テキストの行間は1.8〜2.0を推奨（英語より広め）

- 数値データ（KPI、グラフラベル等）にはJetBrains Monoを使用し、等幅で桁を揃える
- テキスト装飾としてのアンダーラインはリンク以外で使用しない

## 8. 写真・イラストレーション

### 8.1 写真の方向性

| 属性 | 推奨 | 避ける |
|---|---|---|
| トーン | 自然光、落ち着いた色調 | 過度にフィルター加工された画像 |
| 被写体 | 分析作業中のビジネスパーソン、モダンなオフィス | ストックフォト感の強いポージング |
| 構図 | 画面・ダッシュボードが映り込む作業風景 | 握手、指さし等のクリシェ |
| 多様性 | 性別・年齢・民族の多様性を意識 | 均一な属性の偏り |

### 8.2 イラストレーション

- スタイル: フラット＋細いストローク線（Lucideアイコンと同系統）
- カラー: ブランドカラーパレット内に限定
- 用途: LP、オンボーディング、空状態（Empty State）、エラー画面
- 過度にポップ・カートゥーン的なスタイルは避ける（「知的な参謀」としてのトーンを保つ）

## 9. モーション・アニメーション

### 9.1 モーション原則

| 原則 | 説明 |
|---|---|
| **機能的** | アニメーションは装飾ではなく、状態遷移やフィードバックの伝達手段 |
| **控えめ** | 過度なバウンスやスピンは避ける。洗練された印象を保つ |
| **一貫性** | 同じ種類のインタラクションには同じアニメーションを適用 |

### 9.2 イージング・デュレーション

| 種類 | Duration | Easing | 用途 |
|---|---|---|---|
| マイクロ | 100〜150ms | ease-out | ボタンHover、トグル |
| 標準 | 200〜300ms | ease-in-out | モーダル、ドロップダウン |
| 強調 | 300〜500ms | cubic-bezier(0.16, 1, 0.3, 1) | ページ遷移、グラフ描画 |

### 9.3 `prefers-reduced-motion` 対応

アクセシビリティ設定でモーション軽減を選択しているユーザーには、アニメーションを無効化する。

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 10. トーン＆ボイス

### 10.1 ブランドパーソナリティ

PRismは**「知的で頼れる参謀」**として振る舞う。クライアントの隣に座り、データを指差しながら「次はこうしましょう」と提案する存在。

```
┌──────────────────────────────────────────────┐
│                                              │
│   知的 ←─────────────────────── カジュアル    │
│                      ★                       │
│                                              │
│   自信 ←─────────────────────── 控えめ        │
│                ★                             │
│                                              │
│   実用的 ←─────────────────────── 理論的      │
│              ★                               │
│                                              │
│   簡潔 ←─────────────────────── 饒舌          │
│            ★                                 │
│                                              │
└──────────────────────────────────────────────┘
```

### 10.2 ボイス属性

| 属性 | PRismらしい | PRismらしくない |
|---|---|---|
| **知的さ** | データに基づく冷静な分析。専門用語は適度に使用 | 曖昧な精神論。過度に簡略化した説明 |
| **親しみ** | 「です・ます」調。カジュアルだが軽薄ではない | 堅苦しい敬語の羅列。馴れ馴れしいタメ口 |
| **自信** | 「〜です」「推奨します」と言い切る | 「〜かもしれません」「ご参考までに」と逃げる |
| **実用性** | 次の具体的アクションを必ず提示 | 分析結果だけ見せて「ご検討ください」で終わり |
| **簡潔さ** | 1文1メッセージ。冗長な前置きを排除 | 「〜ということもあり、〜を考慮すると…」の連鎖 |

### 10.3 文体の実例

**良い例（PRismらしい）：**
> CLAYGEのInstagram施策を分析しました。Earned比率が競合平均より23%高く、UGC起点の拡散が成功要因です。インフルエンサーとの共創コンテンツを強化することを推奨します。

**悪い例（PRismらしくない）：**
> データを見ると、CLAYGEさんはインスタグラムが結構強いみたいですね！他社さんよりいいねが多い感じなので、頑張ってみてはいかがでしょうか？

### 10.4 ライティングルール

| ルール | 説明 | 例 |
|---|---|---|
| 能動態を使う | 受動態は主体がぼやける | ✅「分析しました」 ❌「分析されました」 |
| 数値を入れる | 具体性がPRismの武器 | ✅「23%高い」 ❌「かなり高い」 |
| 一文一意 | 1つの文に1つのメッセージ | 長い複文を分割する |
| 専門用語は初出で説明 | アクセシビリティの確保 | 「PESO（Paid/Earned/Shared/Owned）」 |

## 15. ブランドの10原則

PRismブランドに関わるすべての判断は、この10原則に立ち返る。

| # | 原則 | 解説 |
|---|---|---|
| 1 | **名前がコンセプトを語る** | 「PRを分光する」という価値が名前に内包されている |
| 2 | **PRの2文字で領域を即伝達** | 誰が見ても「PR関連のツール」と認知できる |
| 3 | **全大文字は絶対に使わない** | 「PRISM」はNSA連想。常に「PRism」表記を死守 |
| 4 | **データに裏打ちされた自信で語る** | 「かもしれない」ではなく「です」のトーン |
| 5 | **分析で終わらず、次の一手を示す** | 示唆を提供することがツールの存在意義 |
| 6 | **モダンで軽い体験を追求する** | 重厚なエンタープライズUIの対極に立つ |
| 7 | **アクセシビリティを妥協しない** | WCAG AA準拠、色覚多様性対応、`prefers-reduced-motion`対応 |
| 8 | **一貫性が信頼を生む** | 全タッチポイントで同じ色・同じ声・同じ体験 |
| 9 | **PESOカラーはブランドの一部** | 4色のPESO区分はPRismの視覚的シグネチャー |
| 10 | **ブランドは進化する** | 年1回のガイドライン見直しで時代に適応する |
